
\chapter{ParaView}

This chapter provides a brief introduction to some of the basic functions in
ParaView. We begin by loading simulation output files and then discuss how to
select the appropriate ParaView data file.

After clicking the \textbf{Open File} button, navigate to the directory where
your data is saved. Dendro commonly outputs data in \texttt{.vtu} format, which
stands for \emph{VTK Unstructured Grid}. For parallel runs, the output is often
organized using a \texttt{.pvtu} file. The \texttt{.pvtu} file acts as a wrapper
that tells ParaView how the individual \texttt{.vtu} pieces fit together.

In most cases, you should open only the \texttt{.pvtu} file rather than opening
the individual \texttt{.vtu} files by hand.

\section{Adjusting the Slice and Viewing the Data and the Calculator}

After selecting the \textbf{Slice} tool, ParaView allows us to choose how the
data should be sliced. Often, we want to create \(z\)-slices, since this is the
default orientation used by Dendro output. To do this, set the slice normal to
\((0,0,1)\). We also set the origin to \((0,0,10^{-5})\). The small nonzero
\(z\)-value is used because setting \(z=0\) can sometimes cause ParaView to miss
the slice.

The \textbf{Show Plane} option should be turned off so that only the data slice
is displayed. After changing these settings, click \textbf{Apply} again.

Once the slice is displayed, we can change which field is visualized using the
field-selection menu. We can also move forward and backward through the
available time steps using the time controls. Finally, the representation menu
allows us to switch between different viewing modes, such as \textbf{Surface}
and \textbf{Wireframe}. These controls are highlighted in
Figure~\ref{fig:paraview-view-controls}.

\section{Color Map Editor}

The \textbf{Color Map Editor} controls how ParaView maps data values to colors.
It can be used to change the color scale, adjust the visible data range, and
choose a different color preset.

\begin{figure}[p]
\centering

\includegraphics[
    width=\textwidth,
    height=0.75\textheight,
    keepaspectratio
]{Paraview/Paraview_7.png}

\caption{The \textbf{Color Map Editor} panel in ParaView. This panel controls
the color scale, opacity, and transfer function used to visualize the selected
field.}
\label{fig:paraview-color-map-editor}

\end{figure}

By default, the color mapping may not automatically update when stepping
through time. The easiest way to ensure that the colors match the data range at
each time step is to enable \textbf{Clamp} and update the color range at every
time step. Otherwise, the coloring may not match the current range of the data
unless it is updated manually.


\section{Macros, making Animations Saving CSV Files, and Finding Data}

Often, we want to perform repetitive tasks in ParaView. These tasks can be
automated by creating a macro. To create a macro, click \textbf{Tools}, then
select \textbf{Start Trace}. Perform the task you want to automate, and when
you are finished, click \textbf{Stop Trace}. ParaView will then allow you to
save the recorded trace as a macro. The macro can later be used to repeat the
same sequence of actions automatically.

We can also save the data currently loaded in ParaView as a CSV file. To do
this, click \textbf{File}, then select \textbf{Save Data}. This will save the
data from the currently active object, such as the current slice. The resulting
CSV file can then be loaded into Python, MATLAB, Mathematica, or another tool
for later analysis.

Animations can also be saved from ParaView by clicking \textbf{File} and then
selecting \textbf{Save Animation}.

Finally, ParaView can be used to find data that satisfies certain conditions.
To do this, click \textbf{View} and then select \textbf{Find Data}. In the
\textbf{Find Data} panel, choose the dataset you want to search. You can usually
leave the element type set to \textbf{Point}. Then choose the field you want to
search, set the desired condition and value, and click \textbf{Find Data}. The
matching region will be highlighted in the render view.
